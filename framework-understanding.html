Playwright Test Framework Understanding

This project is a Java-based Playwright automation framework using JUnit 5 and Page Object Model (POM).

Test Status: Ready (2 tests passed)

1) What This Framework Automates

Demo website: https://www.saucedemo.com/

  • Valid login test checks successful navigation to inventory page.
  • Invalid login test checks expected error message.

2) Framework Design

Base Layer

PlaywrightFactory creates and closes Playwright, Browser, Context, and Page objects.

Utilities Layer

ConfigReader loads values from config.properties.

Page Layer

LoginPage and InventoryPage encapsulate UI interactions and page checks.

Test Layer

LoginTest controls test flow using setup/teardown + assertions.

4) Execution Flow

  1. Test starts and reads baseUrl, username, and password from config.
  2. PlaywrightFactory opens Microsoft Edge in headless mode.
  3. LoginPage performs login action.
  4. InventoryPage validates successful login, or login page validates error text.
  5. Browser context and Playwright are closed in teardown.

6) Key Benefits

  • Clear separation of concerns (Base, Utility, Pages, Tests).
  • Easy maintenance when UI changes (mostly page files need edits).
  • Config-driven credentials and URL for simple environment updates.
  • Scalable foundation for adding more modules and tests.
